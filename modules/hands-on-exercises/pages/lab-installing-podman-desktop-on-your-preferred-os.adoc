#  Lab: Installing Podman Desktop on your preferred OS

= Lab: Installing Red Hat Build of Podman Desktop on Your Preferred OS

== Introduction

This hands-on lab guides you through the process of installing the Red Hat Build of Podman Desktop on your preferred operating system. Podman Desktop provides a user-friendly graphical interface for managing containers, Podman Pods, and Kubernetes resources, simplifying complex container workflows. By the end of this lab, you will have Podman Desktop successfully installed and verified on your system.

=== Objectives

Upon completion of this lab, you will be able to:

*   Identify the necessary prerequisites for Podman Desktop installation.
*   Download and install Red Hat Build of Podman Desktop on macOS, Windows, or Red Hat Enterprise Linux.
*   Perform the initial setup, including Podman machine creation.
*   Verify the successful installation and basic functionality of Podman Desktop.

== Prerequisites

Before you begin, ensure your system meets the following general requirements:

*   **Operating System**: macOS (Intel or Apple Silicon), Windows 10/11 (64-bit), or Red Hat Enterprise Linux 8/9.
*   **System Resources**: At least 4 GB RAM, 2 CPU cores, and 20 GB free disk space recommended for the Podman machine.
*   **Podman Engine**: While Podman Desktop can install a Podman engine, it's beneficial to understand that it manages an underlying Podman engine. On Linux, Podman Desktop leverages the system's Podman installation. On Windows and macOS, it typically sets up and manages a *Podman machine* (a lightweight Linux virtual machine) where the Podman engine runs.

[NOTE]
For Windows users, ensuring that Windows Subsystem for Linux (WSL 2) is enabled and updated is crucial for Podman Desktop to function correctly. Podman Desktop relies on WSL 2 to provision the Podman machine.

== Hands-On Activity: Installing Podman Desktop

Choose the section below that corresponds to your operating system and follow the instructions.

[IMPORTANT]
Do *not* follow the instructions for all operating systems. Select *one* and proceed.

=== Option 1: Installing on macOS

This section details the installation of Red Hat Build of Podman Desktop on macOS systems.

==== Step 1: Download the Installer

. Open your web browser and navigate to the official Red Hat Build of Podman Desktop download page.
. Locate and download the appropriate `.dmg` installer file for your macOS architecture (Intel or Apple Silicon).

==== Step 2: Install Podman Desktop

. Once the download is complete, open the downloaded `.dmg` file.
. A new window will appear, typically showing the Podman Desktop application icon and an "Applications" folder alias.
. Drag the Podman Desktop application icon into the "Applications" folder.
. Close the `.dmg` window and eject the disk image from your Finder sidebar.

==== Step 3: Launch Podman Desktop and Initial Setup

. Open your "Applications" folder and double-click the "Podman Desktop" icon to launch the application.
. The first time you launch Podman Desktop, it might prompt you about downloading from the internet. Click "Open."
. Podman Desktop will perform an initial setup, which includes checking for a Podman engine. If a Podman machine is not detected or created, it will guide you through its creation.
. Follow the on-screen prompts to create a new Podman machine. This process involves downloading a minimal Linux distribution image and setting up a virtual machine where the Podman engine will run. This might take a few minutes.
. Once the Podman machine is successfully created and started, the Podman Desktop dashboard will become accessible.

[TIP]
If you encounter issues during Podman machine creation, ensure you have sufficient disk space and a stable internet connection for downloading the necessary images. Restarting Podman Desktop or your machine can sometimes resolve transient issues.

=== Option 2: Installing on Windows

This section details the installation of Red Hat Build of Podman Desktop on Windows systems, leveraging WSL 2.

==== Step 1: Verify WSL 2 Installation and Update

Podman Desktop on Windows relies heavily on Windows Subsystem for Linux (WSL 2). Ensure it's installed and updated.

. Open PowerShell or Command Prompt *as an administrator*.
. Run the following command to check the WSL version and installed distributions:
+
```powershell
wsl --list --verbose
```
. If WSL 2 is not enabled, or if you need to update it, run:
+
```powershell
wsl --install
wsl --update
```
. You may need to restart your computer after enabling or updating WSL.

[NOTE]
If you don't have any Linux distributions installed for WSL, Podman Desktop will likely guide you through installing a minimal one (like Ubuntu) during its initial setup.

==== Step 2: Download the Installer

. Open your web browser and navigate to the official Red Hat Build of Podman Desktop download page.
. Locate and download the appropriate `.exe` installer file for Windows.

==== Step 3: Install Podman Desktop

. Once the download is complete, double-click the downloaded `.exe` file to start the installer.
. Follow the on-screen instructions. The installation process is typically straightforward, involving accepting the license agreement and choosing an installation directory.
. Allow the installer to complete the process.

==== Step 4: Launch Podman Desktop and Initial Setup

. After installation, launch Podman Desktop from your Start Menu.
. The first time you launch Podman Desktop, it will perform an initial setup, which includes checking for an existing Podman engine or machine.
. Podman Desktop will guide you through creating a new Podman machine within WSL 2 if one doesn't exist. This involves downloading a lightweight Linux distribution and configuring it.
. Follow the on-screen prompts. This process may take several minutes as it downloads and sets up the necessary components.
. Once the Podman machine is successfully created and started, the Podman Desktop dashboard will be displayed.

[WARNING]
If you experience issues with WSL, ensure that virtualization is enabled in your system's BIOS/UEFI settings. Hyper-V might also be required or conflict with other virtualization solutions.

=== Option 3: Installing on Red Hat Enterprise Linux

This section details the installation of Red Hat Build of Podman Desktop on Red Hat Enterprise Linux using `dnf`.

==== Step 1: Ensure `dnf` Configuration and Repositories

Ensure your system has access to the necessary repositories for installing Podman Desktop. Typically, this involves having the `extra-packages-for-enterprise-linux` (EPEL) or similar repositories enabled, though Red Hat may provide it through official channels.

. Open a terminal.
. Update your package cache:
+
```bash
sudo dnf check-update
```

==== Step 2: Install Podman Desktop

Use the `dnf` package manager to install Podman Desktop.

. In your terminal, run the following command:
+
```bash
sudo dnf install podman-desktop
```
. When prompted, type `y` and press Enter to confirm the installation and any required dependencies.
. Wait for the installation process to complete.

==== Step 3: Launch Podman Desktop and Initial Setup

. Once the installation is complete, you can launch Podman Desktop from your applications menu or by typing `podman-desktop` in the terminal and pressing Enter.
. Podman Desktop will launch and automatically detect the local Podman engine installed on your RHEL system.
. It should display the dashboard, showing the active Podman engine (which is your system's Podman). On RHEL, a separate "Podman machine" virtual machine is typically *not* required as the application directly interacts with the host's Podman daemon.

[NOTE]
If you encounter permission issues when Podman Desktop tries to interact with the Podman socket, ensure your user is part of the `podman` group, or configure rootless Podman appropriately.

== Verifying Podman Desktop Installation

After installing Podman Desktop using one of the methods above, perform the following verification steps:

==== Step 1: Launch Podman Desktop

. Open Podman Desktop (if not already open).
. You should see the main dashboard. On the left sidebar, navigate to the "Containers" or "Machines" section.

==== Step 2: Verify Podman Machine Status (macOS/Windows) or Engine Connection (RHEL)

. **macOS/Windows**: In the "Machines" section, you should see your `podman-machine-default` listed and its status as "Running."
. **RHEL**: In the "Settings" (gear icon) -> "Container Engine" section, verify that the "Podman" engine is detected and shows a healthy status.

==== Step 3: Pull and Run a Test Container

Let's run a simple `hello-world` container to confirm everything is working as expected.

. In Podman Desktop, navigate to the "Containers" section.
. Click the "+ Create" button or similar prompt to pull an image.
. In the "Pull an image" dialog, enter `hello-world` as the image name and click "Pull Image."
. Once the image is pulled, it will appear in your "Images" section.
. Go back to "Containers", select the `hello-world` image, and click "Run" (or click the play icon).
. A new container will be created and should run briefly, then exit.
. You can inspect its logs from the container details view to see the "Hello from Docker!" message (this is a common test image).

[TIP]
If the `hello-world` container fails to run, check the logs within Podman Desktop for error messages. Common issues include network connectivity, insufficient resources for the Podman machine, or issues with the Podman machine itself (if on macOS/Windows).

== Troubleshooting Common Installation Issues

*   **"Podman machine not found" or "Podman machine creation failed"**:
    *   **Windows**: Ensure WSL 2 is fully updated and running. Open PowerShell as admin and run `wsl --update` and `wsl --shutdown`. Then retry. Check system BIOS settings for virtualization (Intel VT-x or AMD-V) to be enabled.
    *   **macOS/Linux**: Check for sufficient disk space and RAM. Sometimes, reinstalling the Podman Desktop application might resolve underlying issues.
*   **"Failed to connect to Podman engine"**:
    *   Ensure the Podman machine is started within Podman Desktop.
    *   On Linux, verify the `podman` service is running and accessible to your user. `systemctl --user status podman.socket` or `podman system service --time=0 &`
*   **"Image pull failed"**:
    *   Check your internet connection.
    *   Verify if there are any proxy settings required for your network (see xref:./configuring-and-customizing-podman-desktop.adoc#_setting_up_network_proxies[Configuring network proxies]).
    *   Ensure the image name is correct and the registry is accessible.
*   **Permissions Issues**: On Linux, ensure your user is part of the `podman` group (if applicable for your distribution) or that the `podman` socket has appropriate permissions.

For more in-depth troubleshooting, refer to the xref:./troubleshooting-podman-desktop-issues.adoc[Troubleshooting Podman Desktop Issues] section.
