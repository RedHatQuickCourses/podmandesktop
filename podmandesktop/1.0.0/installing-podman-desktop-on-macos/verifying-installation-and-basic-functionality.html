Verifying installation and basic functionality

Verifying Installation and Basic Functionality

After successfully installing Red Hat Build of Podman Desktop, it’s crucial to verify that all components are functioning as expected. This ensures that the application can properly interact with the underlying container engine and that you can perform fundamental container management tasks. This verification process typically involves checking the Podman Desktop application itself, confirming the status of the Podman engine (and machine, if applicable), and running a simple test container.

Why Verification is Important

Verification serves several key purposes:

  • Confirmation of Setup: It confirms that the installation process completed without errors and that all necessary dependencies are met.

  • Early Problem Detection: Identifies potential issues early on, preventing more complex problems when you start working with advanced container operations.

  • Baseline Functionality: Establishes a baseline of expected behavior, making it easier to troubleshoot future issues by isolating new problems from initial setup failures.

  • Readiness for Use: Ensures that your environment is ready for developing, running, and managing containers.

Understanding the Verification Points

When verifying Podman Desktop, we primarily focus on three areas:

  1. Podman Desktop Application UI: Ensuring the application launches, its dashboard loads correctly, and it detects the container engine.

  2. Podman Engine/Machine Status: Confirming that the underlying Podman engine (and the Podman machine on macOS/Windows) is running, healthy, and accessible by Podman Desktop. This involves checking its CPU, memory, and disk allocation, and ensuring no error messages are displayed.

  3. Basic Container Operations: The ultimate test is to perform a simple container operation, such as pulling an image and running a container. This verifies the end-to-end functionality, including network connectivity, image registry access, and container runtime capabilities.

Lab: Verifying Podman Desktop Installation and Basic Container Operations

This lab guides you through verifying the successful installation of Podman Desktop and performing a basic container lifecycle operation.

Objectives

  • Launch Podman Desktop and navigate its interface.

  • Confirm the status of the Podman engine and any associated Podman machine.

  • Pull a container image from a registry.

  • Run a container based on the pulled image.

  • Verify the container’s execution and logs.

Steps

Open Podman Desktop

Ensure Podman Desktop is launched. * macOS: Search for "Podman Desktop" in Spotlight or find it in your Applications folder. * Windows: Search for "Podman Desktop" in the Start Menu. * Red Hat Enterprise Linux: Search for "Podman Desktop" in your applications menu or run podman-desktop from the terminal if installed via Flatpak or similar.

Check the Dashboard

Once Podman Desktop launches, observe the main dashboard.

  • Expected Outcome: You should see the "Dashboard" view. Look for the "Container Engine" section. It should display a green status indicator (e.g., "Podman is running" or "Podman machine is running") and details about the active engine.

  • Troubleshooting: If it shows "Podman not running" or an error, check the specific error message. You might need to click "Start" next to the engine or investigate the Podman machine status (see next step).

Verify Podman Engine/Machine Status

If you are on macOS or Windows, a Podman machine is typically used. On Linux, Podman runs natively.

On macOS/Windows:

In the "Container Engine" section of the dashboard, you should see the status of your Podman machine (e.g., podman-machine-default).

  • Expected Outcome: The machine should show a "Running" status with allocated CPU, memory, and disk space.

  • Troubleshooting: If the machine is not running, try starting it from the Podman Desktop interface or from a terminal using podman machine start. Consult the "Troubleshooting Podman Desktop Issues" section if problems persist.

On Red Hat Enterprise Linux:

The "Container Engine" section will directly show the status of the local Podman daemon.

  • Expected Outcome: Podman should be indicated as "Running".

  • Troubleshooting: If it’s not running, check your system’s podman service status or restart Podman Desktop.

Pull a Sample Container Image

Now, let’s pull a simple container image to test connectivity and registry access. We will use the ubi8/ubi-micro image from Red Hat’s registry.

  1. Navigate to the Images section In the left-hand navigation pane, click on Images.

  2. Pull the image In the "Images" view, click the Pull an Image button (often an icon resembling a cloud with a downward arrow). A dialog box will appear. In the "Image name" field, enter registry.access.redhat.com/ubi8/ubi-micro. Click Pull.

    • Expected Outcome: The image will start downloading. You should see a progress bar or messages indicating the download status. Once complete, registry.access.redhat.com/ubi8/ubi-micro should appear in your list of local images.

    • Troubleshooting: If the pull fails, check your internet connection, proxy settings (if applicable), and ensure the Podman engine is running correctly. Error messages in the "Pull Image" dialog or the Podman Desktop console might provide more clues.

Run the Sample Container

With the image pulled, let’s run a container from it.

  1. Hover over the ubi8/ubi-micro image in the "Images" list. You will see an action menu appear.

  2. Click the "Run Image" icon (often a play button). A "Create a Container" dialog will appear. You can leave most settings as default for this simple test. Optionally, give it a meaningful name, e.g., my-ubi-test. Click Create.

    • Expected Outcome: The container will start. Podman Desktop will automatically navigate to the Containers section, and you should see your new container listed with a green "Running" status.

    • Troubleshooting: If the container fails to start, check the logs for the container (available by clicking on the container name and then the "Logs" tab). This could indicate issues with the image, container command, or engine.

Verify Container Execution

To confirm the container is truly running and accessible.

  1. Click on the name of your newly created container (e.g., my-ubi-test) in the Containers list. This opens the container’s details view.

  2. Navigate to the Logs tab.

    • Expected Outcome: You should see output in the logs. For ubi8/ubi-micro (which is a minimal base image), you might not see continuous output unless the container is running a specific command. However, the absence of immediate error messages in the logs and a "Running" status confirms basic functionality. If you ran an image like hello-world, you would see its specific output here.

Clean Up (Optional but Recommended)

To keep your environment tidy, stop and remove the test container.

  1. Return to the Containers list.

  2. Hover over your my-ubi-test container.

  3. Click the "Stop" icon (square button) to stop the container.

  4. Once stopped, click the "Delete" icon (trash can) to remove it.

    • Expected Outcome: The container will disappear from the list after being stopped and removed.

Congratulations! You have successfully verified your Red Hat Build of Podman Desktop installation and confirmed its basic ability to manage container images and run containers. You are now ready to explore more advanced features.# Verifying installation and basic functionality

Verifying Installation and Basic Functionality

This section guides you through verifying the successful installation of Red Hat Build of Podman Desktop and performing a basic functional test to ensure your environment is ready for container management.

Technical Explanation

After installing Red Hat Build of Podman Desktop, it’s crucial to confirm that all components are correctly installed and operational. This typically involves:

  1. Launching Podman Desktop: Ensuring the application starts without errors and the graphical user interface (GUI) is accessible.

  2. Podman Machine Status: For macOS and Windows users, verifying that the underlying Podman machine (a virtual machine that hosts the Podman engine) is running and properly connected. This machine provides the Linux environment where containers will actually run.

  3. Container Engine Availability: Confirming that the Podman engine itself is accessible and responsive to commands.

  4. Basic Container Operations: Performing a simple operation, such as pulling a container image and running a basic container, to validate the end-to-end functionality of your setup. This confirms that Podman Desktop can interact with the engine, pull from registries, and execute containers.

If any of these verification steps fail, it indicates a potential issue that needs troubleshooting. Common problems include firewall restrictions, incorrect WSL2 setup (on Windows), or insufficient system resources for the Podman machine.

Hands-On Activity: Verifying Podman Desktop and Running Your First Container

In this lab, you will verify the successful installation of Podman Desktop and deploy a simple 'hello-world' container to confirm basic functionality.

Objective: * Confirm Podman Desktop launches successfully. * Verify the Podman machine is running (if applicable). * Run a basic container and view its output.

Estimated Time: 10-15 minutes

Step 1: Launch Podman Desktop

  1. Locate and launch Podman Desktop:

    • macOS: Find "Podman Desktop" in your Applications folder or via Spotlight search.

    • Windows: Find "Podman Desktop" in your Start Menu.

    • Red Hat Enterprise Linux: Find "Podman Desktop" in your applications menu or launch from the terminal using podman-desktop.

  2. Observe the initial setup:

    • Upon first launch, Podman Desktop might prompt you to set up a new Podman machine or detect an existing one. If prompted, proceed with the default options to create a new machine. This process might take a few minutes as it downloads and configures the necessary virtual machine.

    • Once the setup is complete, you should see the Podman Desktop dashboard.

macOS and Windows

You should see an "Engine is running" status, typically for the podman-machine engine.

Red Hat Enterprise Linux

You should see an "Engine is running" status for the Podman engine, indicating it’s using your host’s native Podman installation.

If you encounter issues like "Podman machine not found" or "Engine not running," consult the "Troubleshooting Podman Desktop Issues" section later in this training.

Step 2: Verify Podman Machine Status (macOS and Windows only)

This step is primarily for macOS and Windows users who rely on the Podman machine. RHEL users can skip this step as Podman runs natively.

  1. Navigate to Settings: In Podman Desktop, click the Settings icon (gear icon) in the bottom-left corner.

  2. Select "Podman Machine": In the settings menu, click on "Podman Machine" on the left sidebar.

  3. Check Status: Verify that your Podman machine is listed and its status is "Running".

If the machine is not running, you can click the Start button to attempt to start it.

Step 3: Run Your First Container

Now, let’s pull a simple hello-world container image and run it.

  1. Return to the Dashboard: Click the Dashboard icon in the left navigation bar.

  2. Open the "Containers" view: In the dashboard, click on the Containers option in the left navigation panel.

  3. Pull an Image:

    • Click the btn:[+ Create] button in the top-right corner.

    • Select "Pull an Image".

    • In the "Image name" field, type docker.io/library/hello-world.

    • Click btn:[Pull image].

      You will see the image being pulled. Once complete, it will appear in your "Images" list.

  4. Run the Container:

    • Navigate to the Images view from the left navigation panel.

    • Find the hello-world image you just pulled.

    • Hover over the image and click the Run button that appears.

      image::podman-desktop-run-hello-world.png[Run Hello-World Container, width=800]

  5. Observe Container Output:

    • A "Run Image" dialog will appear. You can leave the default settings.

    • Click btn:[Run].

    • Podman Desktop will create and run a new container from the hello-world image.

    • Navigate back to the Containers view. You will see your hello-world container listed, likely in a "Exited" state.

    • Click on the hello-world container entry to view its details.

    • In the container details page, click on the Logs tab.

      You should see output similar to this:
      ```text
Hello from Docker!
This message shows that your installation appears to be working correctly.
      To generate this message, Docker took the following steps:
 1. The Docker client contacted the Docker daemon.
 2. The Docker daemon pulled the "hello-world" image from Docker Hub.
    (arm64v8)
 3. The Docker daemon created a new container from that image which runs the
    executable that produces the output you are currently reading.
 4. The Docker daemon streamed that output to the Docker client, which sent it
    to your terminal.
      To try something more ambitious, you can run an Ubuntu container with:
 $ docker run -it ubuntu bash
      Share images, automate workflows, and more with a free Docker ID:
 https://hub.docker.com/
      For more examples and ideas, visit:
 https://docs.docker.com/get-started/
```
      TIP: Although the `hello-world` image shows "Hello from Docker!", it's running perfectly fine on your Podman engine, demonstrating interoperability.

Step 4: Clean Up (Optional)

  1. Stop the Container: (If the container is still running, hello-world usually exits immediately)

    • In the Containers view, select your hello-world container.

    • Click the Stop button (square icon).

  2. Delete the Container:

    • In the Containers view, select your hello-world container.

    • Click the Delete button (trash can icon).

  3. Delete the Image:

    • Navigate to the Images view.

    • Find the hello-world image.

    • Click the Delete button (trash can icon) next to it.

Congratulations! You have successfully verified the installation and basic functionality of Red Hat Build of Podman Desktop by launching the application, checking the engine status, and running your first container. Your environment is now ready for further exploration.
